AdaSoft Deploy Tool

Setup Guide - คู่มือการติดตั้งและใช้งาน

กลับหน้าหลัก

สิ่งที่ต้องเตรียมก่อนใช้งาน

1. ติดตั้ง Git

Deploy Tool ต้องการ Git เพื่อดึงข้อมูล commits และไฟล์ที่เปลี่ยนแปลง

Windows:

  1. ดาวน์โหลด Git จาก: https://git-scm.com/download/win
  2. ติดตั้งตามขั้นตอน (ใช้ค่า default ได้เลย)
  3. สำคัญ: เลือก "Git from the command line and also from 3rd-party software"
  4. ตรวจสอบว่าติดตั้งสำเร็จ: เปิด Command Prompt แล้วพิมพ์
git --version

ควรเห็นเลขเวอร์ชัน เช่น git version 2.43.0

Linux/Mac:

# Ubuntu/Debian
sudo apt-get install git

# CentOS/RHEL
sudo yum install git

# macOS (ถ้ายังไม่มี)
brew install git

2. ตั้งค่า Git Config

ถ้าเป็นครั้งแรกที่ใช้ Git บนเครื่อง ต้องตั้งค่า user:

git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

3. Clone โปรเจ็ค

Clone โปรเจ็คที่ต้องการ deploy ลงในเครื่อง:

# ตัวอย่าง
cd C:\Projects
git clone https://github.com/your-org/AdaPos.git
git clone https://github.com/your-org/AdaStoreBack.git

4. ตั้งค่า Deploy Tool

4.1 แก้ไขไฟล์ includes/config.json

{
    "sourcePath": "C:/Projects/",
    "deployPath": "C:/Deploy/",
    "projects": {
        "AdaPos": "AdaPos/",
        "AdaStoreBack": "AdaStoreBack/"
    },
    "googleDrivePaths": {
        "AdaPos": {
            "xPatch": "https://drive.google.com/drive/folders/YOUR_FOLDER_ID",
            "xUpgrade": "https://drive.google.com/drive/folders/YOUR_FOLDER_ID",
            "xDatabase": "https://drive.google.com/drive/folders/YOUR_FOLDER_ID"
        }
    }
}

4.2 ปรับ Path ให้ถูกต้อง

  • sourcePath: โฟลเดอร์ที่เก็บ source code ที่ clone มา
  • deployPath: โฟลเดอร์ที่จะสร้าง deploy package
  • projects: ชื่อโปรเจ็คและ path ย่อยจาก sourcePath
สำคัญ: ใช้ forward slash (/) แทน backslash (\) เสมอ!

Troubleshooting

ปัญหา: "No git history found" หรือไม่เห็น commits

วิธีแก้:

1. ตรวจสอบว่า Git ติดตั้งแล้ว

git --version

ถ้าไม่เจอ ให้ติดตั้ง Git ก่อน (ดูข้างบน)

2. ตรวจสอบว่า path ถูกต้อง

  • เปิดไฟล์ includes/config.json
  • ตรวจสอบว่า sourcePath + projects.AdaPos รวมกันแล้วเป็น path ที่ถูกต้อง
  • ลองเปิด Command Prompt แล้วพิมพ์:
cd C:\Projects\AdaPos
git log -5

ถ้าเห็น commits แสดงว่า path ถูกต้อง

3. ตรวจสอบว่าเป็น Git repository

cd C:\Projects\AdaPos
git status

ถ้าขึ้น error "not a git repository" แสดงว่าโฟลเดอร์นี้ยังไม่ได้ clone จาก Git

4. ตรวจสอบว่า Git อยู่ใน PATH

  • เปิด Command Prompt แล้วพิมพ์: where git
  • ควรเห็น path เช่น C:\Program Files\Git\cmd\git.exe
  • ถ้าไม่เจอ ให้เพิ่ม Git เข้า System PATH:
    1. Control Panel → System → Advanced system settings
    2. Environment Variables → System variables → Path → Edit
    3. เพิ่ม: C:\Program Files\Git\cmd
    4. Restart Apache/IIS

5. Restart Web Server

หลังจากติดตั้ง Git หรือแก้ไข config ให้ restart web server:

# Apache
httpd -k restart

# หรือ restart service ใน Services.msc

ปัญหา: "Project path not found"

วิธีแก้:

  1. ตรวจสอบ path ใน config.json
  2. ตรวจสอบว่าใช้ forward slash (/) ไม่ใช่ backslash (\)
  3. ตรวจสอบว่าโฟลเดอร์มีอยู่จริง
ตัวอย่างที่ถูกต้อง: 
"sourcePath": "C:/Projects/",
"projects": {
    "AdaPos": "AdaPos/"
}
ตัวอย่างที่ผิด: 
"sourcePath": "C:\\Projects\\",  // ❌ ใช้ backslash
"projects": {
    "AdaPos": "AdaPos"  // ❌ ขาด trailing slash
}

Quick Checklist

ก่อนใช้งาน Deploy Tool ตรวจสอบว่า:

  • ติดตั้ง Git แล้ว (git --version ใช้งานได้)
  • Clone โปรเจ็คลงเครื่องแล้ว
  • แก้ไข includes/config.json ให้ path ถูกต้อง
  • ใช้ forward slash (/) ใน path ทั้งหมด
  • Web server มีสิทธิ์เข้าถึงโฟลเดอร์ source code
  • Git อยู่ใน System PATH
  • Restart web server แล้ว

ยังแก้ไม่ได้?

ส่งข้อมูลเหล่านี้มาให้ทีมช่วยดู:

  1. ผลลัพธ์จาก Command Prompt: 
    git --version
where git
cd C:\Projects\AdaPos
git log -5
  2. ไฟล์ config.json (ส่วน sourcePath และ projects)
  3. Error message จาก Browser Console (F12)
  4. Screenshot หน้าจอที่เกิดปัญหา
